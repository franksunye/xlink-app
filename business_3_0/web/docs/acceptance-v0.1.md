# Business 3.0 Web v0.1 手动验收脚本

> 对应 [10-release-and-acceptance.md](../../docs/10-release-and-acceptance.md) §4 第 3 条；与 [09-v0.1-scope.md](../../docs/09-v0.1-scope.md)「成功标准」及 [flows/](../../docs/flows/) 主路径一致。执行环境：**本地** `npm run dev` 或 **已部署 HTTPS**（Vercel Production / Preview）。

## 前置

1. 在仓库根目录执行 `cd business_3_0/web && npm install`（若未装依赖）。
2. 本地：`npm run dev`，浏览器打开 `http://localhost:3000`。

## 步骤（12～16 步）

1. **未登录门禁**：直接访问 `/work-orders`，应被重定向到 `/login`（URL 可带 `from` 参数）。
2. **登录启动页**：打开 `/login` 应先看到品牌/信任/工人场景区；约 **1.2s** 后自动进入「选择登录方式」，或点「进入登录」立即进入。
3. **登录方式**：应可见 **验证码登录（推荐）**、**密码登录**、**快捷登录** 三卡片；点「验证码登录」进入表单；返回 ‹ 回到方式选择。
4. **登录失败（协议）**：在表单不勾选协议点「登录」，应提示需同意协议，**不**进入首页。
5. **登录失败（桩）**：手机号填 `error`（大小写不敏感），应登录失败且不进入首页。
6. **验证码登录成功**：勾选协议，手机号非 `error`，可点「获取验证码」预填演示码，点「登录」进入 **工作台** `/`。
7. **密码登录路径**：回到登录方式（重新打开 `/login?step=methods` 或退出后），选「密码登录」，输入任意非空密码，勾选协议后应可登录（Mock）。
8. **工作台**：默认 **方案 A（任务优先）**：首屏内容区以「今日任务 → 今日结果 → 快捷操作」为主；进入 **我的**，在「首页方案」中选 **方案 B：数据概览**，返回首页后应以「我的任务 → 今日关键数据 → 紧凑快捷操作」为主（与概念版 `homeVariant` 一致）。
9. **工作台跳转**：在任一方案下，点击跳转到任务列表的入口（若带 `filter`）应进入 `/work-orders?filter=…` 且列表与筛选一致。
10. **任务列表**：在 `/work-orders` 浏览列表；点某一任务进入 `/work-orders/[id]`，可见摘要、当前任务、任务上下文等；在 **跟进记录** Tab 内可见跟进列表，**时间线**亦在该 Tab 底部（与微信小程序一致，无独立底部时间线区块）。
11. **任务 404**：将地址栏 id 改为不存在的 id，应出现「未找到」或等价错误态，并可返回列表。
12. **项目列表**：通过底部 Tab「项目」进入 `/projects`，可见项目卡片与进度条；加载中应有骨架、失败可重试。
13. **我的 / 退出**：在 `/account` 可切换首页方案；点「退出登录」后回到 `/login`；再访问 `/` 应再次要求登录。
14. **401 数据**：清除站点 Cookie 后，在**新标签**直接请求受保护 API（或对 `/api/dashboard` 发未带 Cookie 的请求），应返回 **401**。
15. **（可选，线上）HTTPS**：在已部署环境重复关键步骤，确认地址栏为 **HTTPS**。
16. **（可选）响应头 / SW**：页面应有基础安全头；`/api/*` 的 `Cache-Control` 为 `private, no-store, must-revalidate`；生产环境 SW 对 `/api/*` 为 network-only。

验收通过：以上 **1～14** 无阻塞缺陷；**15～16** 在适用环境执行并通过。
