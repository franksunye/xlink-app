# Web v0.2.2 范围 — 工单详情 Cloud 读（context + activities）

> **状态**：已完成（v0.2.2，2026-05-16）  
> **前置**：[14-v0.2.1-list-mapping.md](./14-v0.2.1-list-mapping.md)（列表映射与 `buildWorkOrderFromSa`）  
> **原则**：详情与列表共用同一套 SA→`WorkOrder` 映射；仅额外推导 `context` 与拉取 `activities`。

---

## 1. 目标

1. **当前任务 / 任务上下文**：由 `status`、`describe`、`applyTimeStr`、当前 `workflowNode` 推导，替换 POC 占位文案。  
2. **跟进记录 Tab**：`biz/action/query.do` 动态列表，与 business 小程序「动态」同源。  
3. **列表与详情一致**：`taskType`、`statusText`、`address`、`tags` 仍走 `buildWorkOrderFromSa`；详情补充 `nodeDef.name`（与列表 Flip 行一致）。

**范围外**：写路径、`orderInfo` / `materials` 深化、距离/同小区、地址补全省市。

---

## 2. 数据流

| 步骤 | API / 模块 | 说明 |
| --- | --- | --- |
| 1 | `loadCodeLabels` | `i/code/selectCode.do`，进程内缓存 |
| 2 | `queryById/{id}.do` `type=query` | `serviceAppointment` + `workflowNode` |
| 3 | `currentNodeDefName` | 第一个 `state !== 10` 的 `nodeDef.name` |
| 4 | `buildWorkOrderFromSa` | 与列表同套字段 |
| 5 | `resolveDetailContext` | 覆盖 `context` |
| 6 | `cloudFetchActivities` | `biz/action/query.do` → `activities` |

BFF：`GET /api/work-orders/[id]` 契约不变；响应头 `X-Xlink-Read-Source: cloud` 时即为真数据。

## 3. context 映射（`sa-detail-context.ts`）

| 字段 | 规则 |
| --- | --- |
| `title` | `taskType`（与列表主标题一致） |
| `desc` | 优先 `describe`；否则按 status 模板句 |
| `badge` | 有 `applyTimeStr` →「已预约」；104 无预约 →「未预约」；否则 `statusText` 或模板 |
| `contextTitle` / `contextDesc` | 有预约 →「已预约时间」+ 格式化时间；104 无预约 →「暂无预约时间」+ 引导；其它 → `statusText` + `describe` 摘要 |
| `primary` / `secondary` | 按 status 只读 CTA（如 104→联系客户/新增预约） |
| `contextIcon` | `iconForTaskType(taskType)` |
| `tone` | 与卡片 `resolveCardTone` 一致 |

## 4. activities 映射（`work-order-cloud-read.ts`）

**请求**

```
POST …/biz/action/query.do
bizType=serviceAppointment
sid={serviceAppointmentId}
actionType=1,2,3,4,5,6
```

**排序**：按 `sortTime` / `createTime` **降序**（最新在上，对齐后端 `ActionServiceImpl` 与小程序展示习惯）。

| `type` | `title` | `badge` | icon / tone |
| --- | --- | --- | --- |
| 1 | 跟进笔记 | 笔记 | ☎ / blue |
| 2 | `name` | 系统 | ▦ / gray |
| 3 | 通话记录 | 电话 | ☎ / blue |
| 4 | 施工日志 | 施工 | ♟ / green |
| 5 | 调度记录 | 调度 | ▦ / orange |
| 6 | 跟进记录 | 跟进 | ☎ / blue |

- `id` ← `typeId`；`content` ← `content`（type 4 可回退 `data`）；`owner` ← `createUserName`；`time` ← `createTime`  
- type 1：`mediaContent` → `photos[]`（URL 列表）

失败或空数组：详情展示既有空态，**不** mock-fallback 整单。

## 5. 可选增强

`buildWorkOrderFromSa(..., { includeCustomerInfo: true })` 填充 `customerInfo`：姓名、地址、工单编号、电话（有则填，无则「—」）。
