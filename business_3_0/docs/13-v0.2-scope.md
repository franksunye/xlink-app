# Web v0.2 范围（POC 波次 2）

> **状态**：已发布（Git tag `v0.2.0`；大项以本快照为准，修正走 patch 或 v0.3 新范围文件）  
> **日期**：2026-05-14  
> **前置真源**：[12-domain-glossary.md](./12-domain-glossary.md)（对象与字段语义）；[09-v0.1-scope.md](./09-v0.1-scope.md) 仅描述 v0.1，**不**承载 v0.2 条目。

## 1. 目标

在 v0.1（Next + BFF + Mock、管家主路径、缓存/SW 基线）之上，完成 **波次 2** 最小竖切：**一条 cloud 只读链路**经 **Adapter + BFF** 进入 Web，并在工单详情增加与现网工单语义方向一致的 **流程节点 / 阶段只读占位**；默认部署 **不因未配置 cloud 而失败**（Mock 与降级路径可验收）。

## 2. 与 v0.1 的关系

| 维度 | v0.1 | v0.2 |
| --- | --- | --- |
| 数据 | 全 Mock | Mock 为主；可选 **cloud 只读**（环境开关） |
| BFF | Mock 直出 | 工单列表/详情读路径经 **统一 Adapter 抽象** |
| UI | 列表/详情主路径 | 详情增加 **只读节点/阶段** 区块（不接完整表单引擎） |

## 3. 范围内

### 3.1 读链路（波次 2 核心）

1. **选定 cloud 只读族（主联调对象）**：与 cloud_ui `WorkOrders.vue` 一致：**`POST`** **`basic/workOrder/query.do`**，`Content-Type: application/x-www-form-urlencoded`，分页参数 `page` / `rows` / `sortField` / `sortOrder` 及与列表页一致的 **`in:state|integer#and=1`**；响应体为 **`FlipInfo`** JSON（根级 **`data`** 为工单数组）。  
2. **同一 Adapter 内、详情必需子路径**：**`POST`** **`basic/workOrder/findById.do`**，body `id=<工单_id>`；响应为 **`ReturnStatus`**（`status===1` 时 **`data.workOrder`** + **`data.workflowNode`**）。仍属 **只读**，不写库。  
3. **Web 工程**：在 `business_3_0/web/` 增加 **读模型 Adapter**（Mock 实现 + Cloud 实现），由 **`USE_CLOUD_READ`** 与 **`XLINK_CLOUD_READ_BASE_URL`** 控制；**Base URL 须与目标前端 API 根一致**——`cloud_ui` 常见 **`…/api`**；**business / beta** 为 **`https://xlinkbeta.fsgo365.cn/fsgo/wm`**（无尾斜杠）。关闭或未配置 base URL 时 **仅 Mock**。  
4. **BFF**：`/api/work-orders` 与 `/api/work-orders/[id]` 在启用 cloud 时经 Adapter 拉取；失败或非 2xx 时 **降级为 Mock**，响应头 **`X-Xlink-Read-Source: mock-fallback`**。鉴权与入口一致时须满足其一：**`X-Auth-Token`**（cloud_ui，环境变量 `XLINK_CLOUD_READ_AUTH_TOKEN` 或请求头 `x-xlink-cloud-token` / `localStorage xlink_cloud_read_token`）或 **`JSESSIONID`**（business / `WeixinMiniServiceServlet`，环境变量 `XLINK_CLOUD_READ_JSESSIONID` 或请求头 `x-xlink-jsessionid` / `localStorage xlink_cloud_read_jsession`；值来自 `index/phoneLogin.do` 的 `data.token`，见 `code/app/business/scripts/api_client.py`）。**Cookie** 头会与上述合并（`JSESSIONID` 可覆盖/追加）。  
5. **契约**：BFF 对外 JSON 仍使用 [12](./12-domain-glossary.md) 约定字段（`WorkOrder`、`activities` 等）；cloud 字段仅在 Adapter 内映射，**禁止**页面直接使用 cloud 原始键名。

### 3.2 UI 壳（与 [08-milestones](./08-milestones.md) 切片 2 对齐）

- 工单详情页在首屏信息区下方增加 **「流程节点（只读）」** 占位区：展示 `readonlyWorkflowNodes`（Mock 为静态演示；cloud 为 `findById` 映射的简化节点序列）。  
- **不接**完整低代码表单引擎、不接节点表单提交。

### 3.3 非功能

- 沿用 [07-nonfunctional-and-poc.md](./07-nonfunctional-and-poc.md)：新增/变更 BFF 路径仍为 **`Cache-Control: private, no-store`**；不扩大 SW 对 `/api` 的缓存面。

## 4. 范围外

- 支付、签约、电话等 **真实外部集成**。  
- cloud **写路径**（派单、改状态、归档等）。  
- **`/projects/[id]`** 项目详情路由（与 v0.1 决策一致，延续不写）。  
- 完整 FSM 配置台、流程编排编辑。

## 5. 成功标准（验收分档）

| 档位 | 条件 | 说明 |
| --- | --- | --- |
| **A — 接 cloud** | `USE_CLOUD_READ=1`、Base URL 正确，且具备 **目标栈鉴权**（`X-Auth-Token` 和/或 **`JSESSIONID`**） | 列表/详情 **`X-Xlink-Read-Source: cloud`**（**含列表为空**：只要 HTTP 契约成功即视为走 cloud，而非 mock-fallback）；详情 **只读节点区** 有数据时与 `findById` 映射一致 |
| **B — 契约完成（Mock-only）** | 未开 cloud 或 cloud 不可用 | 行为与 v0.1 一致 + **Adapter 代码路径在 CI 可构建**；详情仍展示 **静态只读节点演示**；无白屏 |

**出口**：至少 **B** 全过；**A** 在 dev 就绪后由联调同学在同一验收脚本上勾选（见 [web/docs/acceptance-v0.2.md](../web/docs/acceptance-v0.2.md)）。

## 8. cloud 前置与风险

- **Base URL** 必须与入口前端一致：`cloud_ui` 常见 `…/api`；**business / beta** 为 **`https://xlinkbeta.fsgo365.cn/fsgo/wm`**（见 `code/app/business/docs/architecture/env-topology.md`）。  
- **鉴权**：`cloud_ui` 用 **`X-Auth-Token`**；**business 小程序栈**用 **`Cookie: JSESSIONID=<phoneLogin.data.token>`**（可无 `X-Auth-Token`）。BFF 支持两套；联调脚本见 **`npm run verify:cloud-read`**。  
- **业务对象注意**：`business` 冒烟主链路的「工单列表」走的是 **`basic/serviceAppointment/querySAWorkflowNode`**（服务工单）；本 v0.2 读链路按范围文档固定为 **`basic/workOrder/query.do`**（**非服务** `WorkOrderController`）。beta 上默认测试账号下 **后者列表可能为空**，属数据形态差异，**不**表示未连通；可用 `api_inspect.py basic/workOrder/query.do` 与 **`verify:cloud-read`** 验证 HTTP 与鉴权。  
- 未就绪时 **不阻塞** 发版，以 **档位 B** 验收。  
- **Vercel**：平台 **`VERCEL=1`** 时，若未显式关闭（`XLINK_CLOUD_VERCEL_DEFAULTS=0`），BFF 默认 **开启 cloud 读** 且 Base 指向 **beta** `https://xlinkbeta.fsgo365.cn/fsgo/wm`；仍须在 Vercel 配置 **`XLINK_CLOUD_READ_JSESSIONID`** 或 **`XLINK_CLOUD_READ_AUTH_TOKEN`**（或浏览器转发）才能真正拉到数据，否则 **mock-fallback**。详见 `web/README.md` Vercel 节与 `web/.env.example`。  
- Mock 登录与 cloud 会话 **分离**时，若未配置 token / JSESSIONID，将 **mock-fallback**，属预期。
