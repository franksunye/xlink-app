# 低代码拆解、领域模块、外部集成与 POC 波次

> 2026-05-15：把产品拆解思考与 **POC 波次** 对齐，便于排期与依赖管理。技术栈与脱离策略仍以 [05-integration-strategy](./05-integration-strategy.md)、[02-vision](./02-vision.md) 为准。

## 1. 低代码平台：拆解与简化

| 原能力 | 3.0 方向 | 与 POC 的关系 |
| --- | --- | --- |
| **工作流**（原说明驱动：阶段、状态） | **机器可读 spec** 为真源；Web 用类型化页面 + BFF 映射，不再依赖大而全的「运行时元数据渲染」 | **波次 1**：管家竖切只覆盖 spec 中声明的少量状态与节点壳 |
| **自定义表单与表单内逻辑** | **关键路径定制开发**；通用 DSL 不复制 | 随工单节点逐个加屏，不接全量表单引擎 |
| **单据字段扩展** | 在 **3.0 契约与 schema** 内版本化扩展 | 与 Adapter 出参模型一起演进 |
| **报表块**（原不完整） | **重做定制** | **波次 3+**：主链路与结算稳定后再立项 |

## 2. 领域业务模块（与工作流解耦）

与「任务/工单状态机」并行演进、接口上通过 BFF **聚合**：

| 模块 | 说明 | 建议波次 |
| --- | --- | --- |
| **产品 / 报价** | 与下单、签约前链路强相关 | 波次 2 起，与真实或沙箱 API 规划绑定 |
| **结算** | 回款、对账 | 波次 3+ |
| **调度** | 派工、资源排期 | 波次 2～3（依业务优先级） |

## 3. 外部集成

| 集成 | 风险 | 建议 |
| --- | --- | --- |
| **支付** | 高 | Mock → 沙箱 → 生产；单独竖切 + 合规评审 |
| **签约** | 高 | 同上 |
| **电话** | 中高 | 能力占位 + 与现网通话记录/活动类数据对齐时再接 |
| **客服** | 中 | 可晚于支付/签约 |

## 4. POC 波次（与 [08-milestones](./08-milestones.md) 一致方向）

| 波次 | 目标 | 主要内容 |
| --- | --- | --- |
| **1** | 证明 **Next + Vercel + SPA PWA + BFF(A) + 缓存基线** | 登录 Mock、工作台、任务列表/详情、项目列表；`GET` BFF + `Cache-Control` + 客户端 stale；SW 默认不缓存 `/api/*` |
| **2** | 证明 **spec 驱动 + Adapter 接旧 cloud 的一条读链路** | 选定只读接口；契约测试；与 `business` 行为对照 |
| **3** | **写路径与单关键节点表单** + 产品/报价触点之一 | 幂等与错误模型；支付/签约之一进沙箱（若排期允许） |
| **4+** | 结算、调度、报表、外部集成补全 | 按依赖图拉 |

## 5. 管理端 cloud_ui

- **不急**：现网有旧版可用；3.0 **首期聚焦** Web 服务商侧（与 [IA](./04-information-architecture.md) 一致）。  
- **cloud_ui 进 3.0** 作为单独波次或独立应用，待 Web 主路径与 `cloud` 契约稳定后再排。
