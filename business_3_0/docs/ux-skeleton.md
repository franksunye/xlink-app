# Business 3.0 UX Skeleton

## 目标

在不接入后端的前提下，先形成可以运行、可以上传微信体验版、可以多人反馈的前端骨架。

## 第一阶段页面

- 工作台：今日状态、重点事项、常用入口。
- 工单：列表、搜索、详情、处理动作占位。
- 项目：项目进度、负责人、风险提示。
- 我的：体验账号、版本、数据来源。
- 反馈：本地保存体验反馈，后续可替换为后台接口。

## 技术策略

- 新目录独立运行，降低对现有 `business` 的回归风险。
- API 访问通过 `services/api/index.js` 统一出口，当前仅 mock。
- 样式先使用轻量 token，不引入第三方 UI 库。
- 微信小程序 AppID 使用专用体验 AppID，避免误发线上小程序。

## 后续接入点

Spec 到位后再决定：

- 是否拆分分包。
- 是否引入组件库。
- 是否接入 beta 后端或单独的 prototype API。
- 是否需要埋点、远程反馈或用户分组。
