# Web v0.2.3 — 真实登录 + 读贯通

## 目标

在 **同一 JSESSIONID 会话** 下完成管家 Web 四条读路径：工作台、任务、项目、我的；消除 Mock 用户与手工配置 `JSESSIONID` 的双轨。

## 范围

| 模块 | 内容 |
|------|------|
| 登录 | BFF 代理 `index/phoneLogin.do`（`method` 1 验证码 / 2 密码） |
| 会话 | `xlink_b3_sess` = JSESSIONID；`xlink_b3_profile` = 用户摘要 JSON |
| 读贯通 | `GET /api/dashboard`、`/api/work-orders`、`/api/work-orders/[id]`、`/api/projects` |
| 我的 | `GET /api/auth/me`；account 展示真实姓名/角色 |
| Mock 档 | `USE_CLOUD_READ=0` 时登录仍为 `mock-ok`（档位 B） |

## 会话设计

| Cookie | 值 | 说明 |
|--------|-----|------|
| `xlink_b3_sess` | `mock-ok` 或 `phoneLogin.data.token` | httpOnly；门禁 + cloud 鉴权 |
| `xlink_b3_profile` | `{ userId, displayName, roleId, roleName }` | httpOnly JSON |

`resolveCloudJSessionId` 优先级：**登录 cookie（非 mock-ok）** → `x-xlink-jsessionid` → `XLINK_CLOUD_READ_JSESSIONID`。

环境变量：`XLINK_CLOUD_TENANT_ID`（默认 `1340835581184723420`）；`XLINK_CLOUD_READ_JSESSIONID` 仅 CI/脚本兜底。

## API

| 路由 | 行为 |
|------|------|
| `POST /api/auth/login` | cloud：`phoneLogin` → Set-Cookie；失败 401；cloud 不可达降级 mock |
| `GET /api/auth/me` | 返回 profile 或 mock 用户 |
| `POST /api/auth/logout` | 清除 sess + profile |
| `GET /api/dashboard` | cloud：Tab totals + primaryTask；头 `X-Xlink-Read-Source` |
| `GET /api/projects` | cloud：`biz/project/queryProject` |

## 成功标准

1. beta 手机号 + 验证码（测试码 `999999`）登录，**无需**配置 `XLINK_CLOUD_READ_JSESSIONID`
2. 工单/工作台/项目 API 返回 `X-Xlink-Read-Source: cloud`（列表可为空）
3. 工作台四格与任务 Tab 计数一致
4. 「我的」显示登录用户
5. 退出后 API 401；档位 B mock 登录仍可用
